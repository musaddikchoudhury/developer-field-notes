Developer Field Notes

Git vs. GitHub

Git is a local version control system that tracks changes to your code over time. GitHub is a cloud-based platform that hosts those Git repositories online, allowing you to back up your work and collaborate with others.

The Daily Workflow

The daily loop consists of: edit, stage, commit, and push.

Git Command Reference

  • git init: Initializes a new local repository.
  • git status: Shows the current state of your working directory and staging area.

Git Command Reference

  • git init: Initializes a new, empty local Git repository.
  • git status: Shows the current state of your files, letting you know what is tracked, modified, or staged.
  • git add .: Stages all of your modified and new files so they are ready to be committed.
  • git commit -m "message": Saves a snapshot of your staged changes to your local history with a descriptive note.
  • git push: Uploads your local commits to a remote repository on GitHub.
  • git pull: Fetches the newest updates from your GitHub repository and merges them into your local files.
  • git diff: Shows the exact lines of code that were added, modified, or removed before you stage them.
  • git log: Displays a history timeline of all the commits made in your project.
